# Security Policy

## Reporting a Vulnerability

If you believe you have found a security vulnerability in this project, please **do not open a public GitHub issue immediately**.

Instead, please report it privately to the project maintainer with:

- a clear description of the issue
- steps to reproduce it
- affected files or modules if known
- potential impact
- any suggested mitigation, if available

If private reporting channels are expanded later, this document will be updated accordingly.

---

## What to Report

Examples of issues worth reporting include:

- unsafe execution paths
- command injection risks
- insecure file handling
- dependency-related vulnerabilities
- sensitive data exposure
- unsafe automation behavior
- weaknesses in verification or execution flows that could cause harmful unintended actions

---

## Response Goals

The project will aim to:

- acknowledge receipt of the report
- investigate the issue
- determine impact and scope
- prepare a fix where appropriate
- disclose responsibly after remediation when possible

---

## Scope

This policy applies to the code and workflows in this repository.

Please report issues responsibly and in good faith.

---

## Notes

This project is still evolving, so some areas may not yet have mature hardening or deployment guidance. Responsible disclosure is appreciated.
